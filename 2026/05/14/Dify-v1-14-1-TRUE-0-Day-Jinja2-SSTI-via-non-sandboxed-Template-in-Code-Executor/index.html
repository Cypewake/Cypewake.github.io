Dify v1.14.1 TRUE 0-Day Jinja2 SSTI via non-sandboxed Template in Code Executor

Overview

A zero-day vulnerability was discovered in Dify (version 1.14.1 and earlier) that allows attackers to achieve Remote Code Execution (RCE) via Server-Side Template Injection (SSTI) in the Jinja2 template engine used by the code executor.

The vulnerability exists because the Jinja2TemplateTransformer class uses jinja2.Template directly without sandboxing, allowing user-controlled template code to execute arbitrary Python commands.

Affected Component

  • File: dify/api/core/helper/code_executor/jinja2/jinja2_transformer.py
  • Class: Jinja2TemplateTransformer
  • Method: get_runner_script (line 47) and get_preload_script (line 71)
  • Vulnerable Code: template = jinja2.Template(template_code)

Vulnerability Details

Dify’s code executor allows users to run custom code via workflows or prompt templates. The Jinja2 template engine is used to process template code before execution. However, the code uses the standard jinja2.Template class instead of the sandboxed jinja2.sandbox.SandboxedEnvironment.

This allows an attacker to inject Jinja2 template syntax that accesses Python builtins, subclasses, and ultimately execute arbitrary system commands.

Expected Output

The script will output the contents of /etc/passwd, confirming file read capability. Similar techniques can be used to execute arbitrary commands via __builtins__.eval or subprocess.

Impact

  • Remote Code Execution: An attacker can execute any system command with the privileges of the Dify service (often root in container deployments).
  • Data Theft: Read arbitrary files on the server (source code, configuration, secrets).
  • Network Attacks: Use the server as a pivot for internal network scanning or further exploitation.
  • Persistence: Install backdoors, create reverse shells, or modify cron jobs.

Attack Vector

An attacker needs the ability to create or edit a workflow or prompt template in Dify. This is typically available to users with workspace.tools or workspace.prompt-template permissions, which are often granted to regular users in multi-tenant Dify instances.

Once the malicious template is saved, triggering the workflow or using the prompt template will execute the SSTI payload.

Vulnerability Verification Steps

  1. Clone the Dify repository (version 1.14.1 or earlier).
  2. Locate the file api/core/helper/code_executor/jinja2/jinja2_transformer.py.
  3. Confirm the use of jinja2.Template without sandboxing.
  4. Run the provided PoC script to verify code execution.
  5. No known CVE or CNVD assigned at the time of discovery (2026-05-14).
